# Tasks Overview

Tasks are generic work items used for follow-ups, QA tasks, or general project work. They are not limited to testing.

## Fields
- Title (required)
- Description (optional)
- Assigned user (optional)
- Due date (optional)
- Status: TODO, IN_PROGRESS, DONE
- Comments

## Behavior
- Tasks are created within a project.
- Tasks can be assigned to any project member.
- Comments are attached to tasks and include author information.
- The dashboard surfaces assigned tasks and completed work.
